The <HT><a href="https://pub.dev/documentation/flutter_reactter/latest/flutter_reactter/RtScope-class.html">`RtScope`</a></HT> widget serves as an entry point for Reactter, allowing you to use any consumer(such as <HT>[`RtConsumer`](/reactter/api/widgets/rt_consumer)</HT>, <HT>[`RtSelector`](/reactter/api/widgets/rt_selector)</HT>, <HT>[`RtComponent`](/reactter/api/widgets/rt_component)</HT>, <HM>[`BuildContext.watch`](/reactter/api/extensions/builder_context_watch)</HM>, <HM>[`BuildContext.select`](/reactter/api/extensions/builder_context_select)</HM>) to observe state changes within the subtree of the widget it wraps, without the need to explicitly define dependencies.

:::caution
The <HT>`RtScope`</HT> widget is optional, but if you use any consumers without defining dependencies and their ancestor widgets are not wrapped by it, a <HT>`RtScopeNotFoundException`</HT> error will be thrown. Therefore, make sure this widget is correctly placed in the widget tree hierarchy.

To avoid such errors, it is advisable to wrap your application's root widget with it.
:::

## Syntax

```dart showLineNumbers=false
RtScope({
  Key? key,
  required Widget child,
})
```

## Properties

- `key`: An optional <HT>`Key`</HT> to use for identifying the widget.
- `child`: The <HT>`Widget`</HT> that is to be wrapped by the <HT>`RtScope`</HT> widget.
