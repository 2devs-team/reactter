Reactter provides several methods to manage dependencies in a more efficient way.
These methods are used to create and manipulate dependencies in a more organized way.

## <HM>`Rt.register`</HM>

The <HM>`Rt.register`</HM> method registers a new dependency.
It returns <HN>`true`</HN> if the dependency is registered successfully, otherwise return <HN>`false`</HN>.

#### Syntax

```dart showLineNumbers=false
bool Rt.register<T>(
  T Function() builder, {
  String? id,
  DependencyMode mode = DependencyMode.builder,
});
```

#### Parameters

- <HM>`builder`</HM>: A function that returns an instance of the <HT>`T`</HT> dependency.
- `id`: An optional identifier for the <HT>`T`</HT> dependency. If not provided, the dependency is registered by its type(<HT>`T`</HT>).
- `mode`: The mode of the dependency. It can be <HE>`DependencyMode.builder`</HE>, <HE>`DependencyMode.factory`</HE>, or <HE>`DependencyMode.singleton`</HE>. 
The default value is <HE>`DependencyMode.builder`</HE>. Learn more about it in [DependencyMode](/reactter/core_concepts/dependency_injection#dependency-modes).

## <HM>`Rt.delete`</HM>

The <HM>`Rt.delete`</HM> method removes the dependency instance and its registration from the Reactter based on the dependency mode.
It returns <HN>`true`</HN> if the dependency is successfully deleted, otherwise return <HN>`false`</HN>.

:::note
The behavior of the <HM>`Rt.delete`</HM> method depends on the dependency mode:
- If the dependency is registered as <HE>Builder</HE> mode, both the dependency instance and its registration will be deleted.
- If the dependency is registered as <HE>Factory</HE> mode, only the dependency instance will be deleted, but its registration remains.
- If the dependency is registered as <HE>Singleton</HE> mode, nothing will be deleted.

For more information about the dependency modes, refer to [Dependency Modes](/reactter/core_concepts/dependency_injection/#dependency-modes).
:::

#### Syntax

```dart showLineNumbers=false
bool Rt.delete<T>([String? id]);
```

#### Parameters

- `id`: An optional identifier for the <HT>`T`</HT> dependency. If not provided, the default instance of the <HT>`T`</HT> dependency is used.
